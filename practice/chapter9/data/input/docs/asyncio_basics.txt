# Python asyncio 기초

## 비동기 프로그래밍이란?

비동기 프로그래밍은 I/O 작업을 기다리는 동안 다른 작업을 수행할 수 있게 해주는 프로그래밍 패러다임이다. 전통적인 동기 프로그래밍에서는 하나의 작업이 완료될 때까지 프로그램이 대기하지만, 비동기 프로그래밍에서는 여러 작업을 동시에 처리할 수 있다.

Python의 asyncio 모듈은 비동기 I/O를 위한 표준 라이브러리다. Python 3.4에서 처음 도입되었으며, Python 3.7에서 async/await 문법이 안정화되었다.

## 코루틴(Coroutine)

코루틴은 async def로 정의되는 함수다. 일반 함수와 달리 실행을 일시 중지하고 재개할 수 있다. await 키워드를 사용하여 다른 코루틴의 완료를 기다릴 수 있다.

```python
async def hello():
    print("Hello")
    await asyncio.sleep(1)
    print("World")
```

코루틴을 호출하면 코루틴 객체가 반환된다. 실제로 실행하려면 이벤트 루프에서 실행해야 한다.

## 이벤트 루프(Event Loop)

이벤트 루프는 비동기 작업을 관리하고 스케줄링하는 핵심 구성 요소다. asyncio.run() 함수를 사용하면 이벤트 루프를 자동으로 생성하고 관리한다.

```python
import asyncio

async def main():
    await hello()

asyncio.run(main())
```

Python 3.7 이전에는 이벤트 루프를 수동으로 생성하고 관리해야 했지만, asyncio.run()이 도입되면서 훨씬 간단해졌다.
