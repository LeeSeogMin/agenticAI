# 제10장: 에이전트 보안과 신뢰성: 검증에서 가드레일까지 — 집필 계획서

## 장 개요

이 장은 LLM 기반 에이전트 시스템에서 환각 방지와 신뢰성 확보를 위한 검증 전략을 다룬다. v2.0에서 OWASP AI Agent Security, 다층 가드레일 아키텍처, 에이전트 평가 프레임워크 섹션을 추가하여 보안과 신뢰성의 전체 그림을 조망한다.

### 학습 목표

1) 환각의 유형과 원인을 이해하고, 교차검증·출처 일치·품질 게이트를 포함한 검증 파이프라인을 설계한다.
2) OWASP가 정의한 에이전트 보안 위협 4가지를 식별하고 대응 전략을 수립한다.
3) 입력·실행·출력·접근 제어로 구성된 다층 가드레일 아키텍처를 설계한다.
4) 에이전트 성능을 정량적으로 평가하는 벤치마크와 프레임워크를 활용한다.

### 장 구성

- 10.1 LLM 환각의 이해
- 10.2 교차검증 전략
- 10.3 출처 일치 검증
- 10.4 품질 게이트 설계
- 10.5 검증 파이프라인 구현
- 10.6 OWASP AI Agent Security [신규]
- 10.7 다층 가드레일 아키텍처 [신규]
- 10.8 에이전트 평가 프레임워크 [신규]
- 10.9 실습: 검증 워크플로우 구현
- 10.10 실패 사례 수집과 개선
- 10.11 실패 사례와 교훈

## 10.6 OWASP AI Agent Security [신규]

### 집필 방향

OWASP가 정의한 에이전트 고유 보안 위협(프롬프트 인젝션, 도구 오용, 데이터 유출, 과도한 자율성)과 대응 전략을 서술한다.

### 강화 항목(반드시 포함)

- 프롬프트 인젝션: 직접/간접 공격 벡터, 입력 검증 방어
- 도구 오용: 권한 최소화, 파라미터 검증
- 데이터 유출: PII 마스킹, 출력 필터링
- 과도한 자율성: 행동 범위 제한, 사람 승인 의무화
- 표 10.2: OWASP 에이전트 보안 위협 4대 분류

---

## 10.7 다층 가드레일 아키텍처 [신규]

### 집필 방향

입력·실행·출력·접근 제어 4개 계층으로 구성된 가드레일 아키텍처를 서술한다. OpenAI Agents SDK, LangGraph, Guardrails AI 등 프레임워크 내장 가드레일도 다룬다.

### 강화 항목(반드시 포함)

- 입력 계층: PII 제거, 탈옥 시도 탐지, 토픽 제한
- 실행 계층: 도구 호출 샌드박싱, 타임아웃/비용 제한
- 출력 계층: 정책 위반 필터링, Faithfulness 점수
- 접근 제어: 역할 기반 인가, 감사 로깅
- 프레임워크 내장 가드레일 비교

---

## 10.8 에이전트 평가 프레임워크 [신규]

### 집필 방향

에이전트 성능을 정량적으로 평가하는 벤치마크와 프레임워크를 서술한다.

### 강화 항목(반드시 포함)

- 표 10.3: 에이전트 벤치마크 비교 (GAIA, WebArena, τ-bench, Terminal-Bench)
- CLEAR 프레임워크: Correctness, Latency, Efficiency, Autonomy, Reliability
- LLM-as-Judge 패턴과 한계
- 평가 자동화 파이프라인 설계
