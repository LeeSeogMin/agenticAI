# 제10장: 신뢰성(환각 방지)과 검증 파이프라인 — 집필 계획서

## 장 개요

이 장은 LLM 기반 시스템에서 환각(hallucination)을 방지하고 신뢰성을 확보하는 검증 전략을 다룬다. 학습자는 (1) 환각의 유형과 원인, (2) 교차검증/출처 일치 전략, (3) 품질 게이트 설계, (4) 실패 시 재시도/사람 승인 워크플로우를 익힌다. 실습에서는 검증 실패 시 재시도와 사람 승인이 포함된 완전한 워크플로우를 구현한다.

### 학습 목표

1) LLM 환각의 유형과 발생 원인을 이해한다.
2) 교차검증과 출처 일치 검증 전략을 학습한다.
3) 품질 게이트와 검증 파이프라인을 설계한다.
4) 검증 실패 시 재시도 및 사람 승인 워크플로우를 구현한다.
5) 실패 사례 수집과 지속적 개선 방법을 익힌다.

---

## 10.1 LLM 환각의 이해

### 집필 방향

환각(hallucination)의 정의와 유형을 설명한다. 환각이 발생하는 원인과 실무에서의 위험성을 구체적인 예시와 함께 다룬다.

### 강화 항목(필수)

- 환각의 정의와 유형 (사실 오류, 논리 오류, 출처 조작)
- 환각 발생 원인 (학습 데이터 편향, 컨텍스트 부족, 토큰 확률 분포)
- 환각의 실무적 위험성
- 환각 탐지의 어려움

---

## 10.2 교차검증 전략

### 집필 방향

동일 질문에 대해 여러 소스나 모델을 활용한 교차검증 방법을 설명한다. Self-consistency, Multi-model verification 등의 기법을 다룬다.

### 강화 항목(필수)

- Self-consistency (동일 모델 다중 샘플링)
- Multi-model verification (다중 모델 교차검증)
- Chain-of-verification (검증 체인)
- 응답 일관성 점수 계산

### 표(권장)

**표 10.1** 교차검증 전략 비교

| 전략 | 방법 | 비용 | 신뢰도 |
|-----|------|-----|-------|
| Self-consistency | 동일 모델 N회 샘플링 | 중간 | 중간 |
| Multi-model | 서로 다른 모델로 검증 | 높음 | 높음 |
| Chain-of-verification | 검증 전용 프롬프트 추가 | 낮음 | 중간 |

## 10.3 출처 일치 검증

### 집필 방향

RAG 시스템에서 생성된 답변이 실제 출처 문서와 일치하는지 검증하는 방법을 다룬다. NLI(Natural Language Inference) 기반 검증과 인용 추적을 설명한다.

### 강화 항목(필수)

- 출처 기반 답변(Grounded Generation)
- NLI 기반 일치성 검증
- 인용 정확성 검사
- Faithfulness 점수 계산

---

## 10.4 품질 게이트 설계

### 집필 방향

생성 파이프라인에 품질 검사 지점(Quality Gate)을 설계하는 방법을 다룬다. 게이트 통과 기준과 실패 시 처리 전략을 설명한다.

### 강화 항목(필수)

- 품질 게이트 개념과 배치 위치
- 게이트 통과 기준 설정 (임계값, 규칙)
- 다단계 품질 검사
- 게이트 실패 시 처리 전략

### 그림(권장)

**그림 10.1** 품질 게이트가 포함된 생성 파이프라인

```
[입력] → [생성] → [Gate 1: 형식] → [Gate 2: 출처] → [Gate 3: 일관성] → [출력]
              ↓ 실패        ↓ 실패          ↓ 실패
           [재시도]      [재시도]       [사람 승인]
```

## 10.5 검증 파이프라인 구현

### 집필 방향

실제 검증 파이프라인을 구현하는 방법을 설명한다. LangChain/LangGraph를 활용한 검증 노드 구성을 다룬다.

### 강화 항목(필수)

- 검증 노드 설계 (형식, 출처, 일관성)
- 검증 결과 구조화
- 검증 실패 로깅
- 재시도 로직 구현

---

## 10.6 실습: 검증 워크플로우 구현

### 실습 목표

1) 교차검증과 출처 일치 검증이 포함된 파이프라인을 구현한다.
2) 검증 실패 시 자동 재시도 로직을 추가한다.
3) 최대 재시도 초과 시 사람 승인 요청 흐름을 구현한다.
4) 검증 결과와 실패 로그를 저장한다.

### 작업 정의

LangGraph를 활용하여:
- 질문 → 답변 생성 → 교차검증 → 출처 검증 → 품질 게이트
- 검증 실패 시 재시도 (최대 3회)
- 재시도 초과 시 사람 승인 요청

### 필요 라이브러리

- `langgraph`: 워크플로우 구성
- `langchain-openai`: OpenAI LLM
- `chromadb`: 벡터 저장소 (RAG)
- `python-dotenv`: 환경 변수

### 실습 코드

- `practice/chapter10/code/10-6-verification.py`: 검증 워크플로우 구현
- `practice/chapter10/code/requirements.txt`: 의존성 목록

### 실습 산출물(필수)

- `practice/chapter10/data/output/ch10_verification_result.json`: 검증 결과
- `practice/chapter10/data/output/ch10_failure_log.json`: 실패 로그

## 10.7 실패 사례 수집과 개선

### 집필 방향

검증에서 발견된 실패 사례를 수집하고 시스템을 지속적으로 개선하는 방법을 다룬다.

### 강화 항목(필수)

- 실패 사례 분류 (유형, 심각도)
- 실패 패턴 분석
- 프롬프트/검색 개선 피드백 루프
- 검증 기준 조정

---

## 10.8 실패 사례와 교훈

### 집필 방향

검증 파이프라인 구현 시 발생할 수 있는 실패 유형과 대응 방법을 다룬다.

### 강화 항목(필수)

- 과도한 검증으로 인한 지연
- False positive/negative 균형
- 검증 비용 최적화
- 사람 승인 병목 현상
