# 제12장: 배포·관측·비용 최적화 — 집필 계획서

## 장 개요

이 장은 AI 에이전트 시스템을 프로덕션 환경에 배포하고 운영하는 방법을 다룬다. 학습자는 (1) 배포 옵션과 고려사항, (2) 관측 가능성(Observability) 구현, (3) 비용 추정과 최적화, (4) 보안과 키 관리를 익힌다. 실습에서는 운영 체크리스트를 작성하고 모니터링 시스템을 구성한다.

### 학습 목표

1) 로컬/컨테이너 기반 배포 옵션을 이해하고 선택 기준을 학습한다.
2) LLM 애플리케이션의 관측 가능성(로깅, 트레이싱, 메트릭)을 구현한다.
3) API 호출 비용을 추정하고 최적화 전략을 적용한다.
4) API 키 관리와 최소 권한 원칙을 실무에 적용한다.
5) 프로덕션 운영 체크리스트를 작성한다.

---

## 12.1 배포 옵션과 고려사항

### 집필 방향

로컬 실행, 컨테이너(Docker), 클라우드 배포의 장단점을 비교한다. 에이전트 시스템에 적합한 배포 전략을 설명한다.

### 강화 항목(필수)

- 로컬 실행: 개발/테스트 환경
- Docker 컨테이너: 이식성과 재현성
- 클라우드 배포: 확장성과 가용성
- 선택 기준: 팀 규모, 트래픽, 예산

### 표(권장)

**표 12.1** 배포 옵션 비교

| 옵션 | 장점 | 단점 | 적합한 상황 |
|-----|-----|-----|-----------|
| 로컬 | 간단, 빠른 피드백 | 확장 불가, 단일 장애점 | 개발, 테스트 |
| Docker | 이식성, 재현성 | 오케스트레이션 필요 | 소규모 프로덕션 |
| 클라우드 | 확장성, 고가용성 | 복잡성, 비용 | 대규모 프로덕션 |

---

## 12.2 관측 가능성 (Observability)

### 집필 방향

LLM 애플리케이션의 관측 가능성 3요소(로깅, 트레이싱, 메트릭)를 설명한다. LangSmith, Langfuse 등 도구를 소개한다.

### 강화 항목(필수)

- 로깅: 입력, 출력, 에러 기록
- 트레이싱: 요청 흐름 추적, 지연 시간 분석
- 메트릭: 처리량, 응답 시간, 성공률
- 도구: LangSmith, Langfuse, OpenTelemetry

---

## 12.3 운영 메트릭 설계

### 집필 방향

LLM 에이전트 시스템에서 추적해야 할 핵심 메트릭을 정의하고 대시보드 구성 방법을 설명한다.

### 강화 항목(필수)

- 핵심 메트릭: 요청 수, 응답 시간, 토큰 사용량, 에러율
- SLI/SLO 정의
- 알림 설정 (임계값 기반)
- 대시보드 구성

---

## 12.4 비용 추정과 최적화

### 집필 방향

LLM API 비용 구조를 설명하고, 비용을 추정/모니터링/최적화하는 방법을 다룬다.

### 강화 항목(필수)

- 토큰 기반 과금 구조
- 비용 추정 공식
- 캐싱 전략 (프롬프트 캐싱, 응답 캐싱)
- 모델 선택 최적화 (작은 모델 우선)
- 프롬프트 최적화 (토큰 절약)
