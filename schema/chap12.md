# 제12장: 배포·관측·비용 최적화와 MCP 운영 — 집필 계획서

## 장 개요

이 장은 AI 에이전트 시스템을 프로덕션 환경에 배포하고 운영하는 방법을 다룬다. v2.0에서 관측성 플랫폼 5종 비교, 설계 시점부터의 비용 최적화 전략, MCP Registry와 서버 발견 섹션을 추가하여 프로덕션 운영의 전체 그림을 조망한다.

### 학습 목표

1) 로컬/컨테이너 기반 배포 옵션을 이해하고, 관측 가능성 플랫폼을 비교·선택한다.
2) 모델 라우팅, 적응형 추론 제어, 프롬프트 캐싱 등 설계 시점부터의 비용 최적화 전략을 적용한다.
3) MCP Registry를 활용하여 프로덕션 환경에서 MCP 서버를 관리하고 발견하는 방법을 설명한다.
4) API 키 관리와 최소 권한 원칙을 실무에 적용한다.

### 장 구성

- 12.1 배포 옵션과 고려사항
- 12.2 관측 가능성 (Observability)
- 12.3 운영 메트릭 설계
- 12.4 비용 추정과 최적화
- 12.5 보안과 키 관리
- 12.6 프로덕션 체크리스트
- 12.7 관측성 플랫폼 비교 [신규]
- 12.8 비용 최적화: 설계 시점부터 [신규]
- 12.9 MCP Registry와 서버 발견 [신규]
- 12.10 실습: 운영 환경 구성
- 12.11 실패 사례와 교훈

---

## 12.7 관측성 플랫폼 비교 [신규]

### 집필 방향

LangSmith, Langfuse, Arize Phoenix, Braintrust, Helicone 5종을 비교하고 선택 기준을 제시한다.

### 강화 항목(반드시 포함)

- 표 12.3: 5대 관측성 플랫폼 비교 (배포 모델, 무료 티어, 핵심 강점)
- 선택 기준: 프레임워크 호환성, 배포 모델, 핵심 관심사

---

## 12.8 비용 최적화: 설계 시점부터 [신규]

### 집필 방향

모델 라우팅, 적응형 추론 제어(effort level), 프롬프트 캐싱, 배치 API를 포함한 고급 비용 최적화 전략을 서술한다.

### 강화 항목(반드시 포함)

- 표 12.4: 작업 복잡도별 모델 라우팅 전략
- 적응형 추론 제어: effort 파라미터 (low/medium/high/max)
- 프롬프트 캐싱: 읽기 비용 10%, 5분/1시간 캐시
- 배치 API: 50% 할인, 비동기 파이프라인

---

## 12.9 MCP Registry와 서버 발견 [신규]

### 집필 방향

프로덕션 환경에서 MCP 서버를 관리·배포·모니터링하는 방법과 MCP 마켓플레이스를 서술한다.

### 강화 항목(반드시 포함)

- MCP Registry 개념: 중앙 디렉터리 서비스
- MCP 마켓플레이스: Smithery, mcp.run
- 프로덕션 관리: 서버 등록·발견, 버전 관리, 모니터링, 보안
