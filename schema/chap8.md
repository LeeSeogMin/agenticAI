# 제8장: RAG의 기본과 프로덕션 체크리스트 — 집필 계획서

## 장 개요

이 장은 검색 증강 생성(Retrieval Augmented Generation, RAG)의 기본 개념과 프로덕션 환경에서의 체크리스트를 다룬다. 학습자는 (1) RAG의 필요성과 아키텍처, (2) 벡터 데이터베이스 활용, (3) 검색 품질 향상 전략, (4) 인용/출처 추적, (5) 비용 최적화를 익힌다. 실습에서는 작은 문서 코퍼스를 대상으로 검색-답변-출처 표시 흐름을 구현한다.

### 학습 목표

1) RAG가 필요한 상황과 아키텍처를 이해한다.
2) 벡터 데이터베이스(ChromaDB)를 사용하여 문서를 임베딩하고 검색한다.
3) 검색 품질을 향상시키는 청킹과 리랭킹 전략을 적용한다.
4) 답변에 인용/출처를 포함하는 방법을 구현한다.
5) 프로덕션 환경의 체크리스트(캐싱, 비용, 평가)를 작성한다.

---

## 8.1 RAG가 필요한 이유

### 집필 방향

LLM의 한계(지식 컷오프, 환각, 도메인 지식 부족)를 설명하고, RAG가 이를 해결하는 방식을 소개한다. 파인튜닝과 비교하여 RAG의 장점을 설명한다.

### 강화 항목(필수)

- LLM만으로 해결하기 어려운 문제 유형
- RAG vs 파인튜닝 비교
- RAG 아키텍처 개요 (인덱싱 → 검색 → 생성)

### 표(권장)

**표 8.1** RAG vs 파인튜닝 비교

| 기준 | RAG | 파인튜닝 |
|-----|-----|---------|
| 지식 업데이트 | 문서 추가로 즉시 | 재학습 필요 |
| 비용 | 임베딩/검색 비용 | 학습 비용 높음 |
| 출처 추적 | 가능 | 불가능 |
| 환각 감소 | 근거 기반 답변 | 제한적 |

---

## 8.2 벡터 데이터베이스와 임베딩

### 집필 방향

텍스트를 벡터로 변환하는 임베딩 개념과, 벡터 데이터베이스(ChromaDB)를 사용한 저장 및 검색 방법을 설명한다.

### 강화 항목(필수)

- 임베딩의 원리 (텍스트 → 벡터)
- 유사도 검색 (코사인 유사도, 유클리드 거리)
- ChromaDB 기본 사용법
- 다른 벡터 DB 선택 기준 (Pinecone, Milvus, Weaviate)

---

## 8.3 청킹 전략: 문서를 효과적으로 분할하기

### 집필 방향

문서를 적절한 크기로 분할하는 청킹 전략을 설명한다. 청크 크기, 오버랩, 의미 기반 청킹 등 다양한 접근법을 비교한다.

### 강화 항목(필수)

- 고정 크기 청킹 vs 의미 기반 청킹
- 청크 크기와 오버랩 설정
- 메타데이터 보존 전략

---

## 8.4 검색 품질 향상: 리랭킹과 하이브리드 검색

### 집필 방향

기본 벡터 검색의 한계를 설명하고, 검색 품질을 향상시키는 기법(리랭킹, 하이브리드 검색)을 소개한다.

### 강화 항목(필수)

- 리랭킹(Reranking)의 개념과 효과
- 하이브리드 검색 (키워드 + 벡터)
- Top-K 조정과 필터링

---

## 8.5 인용과 출처 추적

### 집필 방향

RAG 시스템에서 답변에 출처를 포함하는 방법을 설명한다. 신뢰성 향상과 검증 가능성을 강조한다.

### 강화 항목(필수)

- 청크에서 원본 문서로 역추적
- 답변에 인용 형식으로 출처 포함
- 출처 신뢰도 표시

## 8.6 실습: 문서 기반 Q&A 시스템

### 실습 목표

1) 문서를 청킹하고 ChromaDB에 저장한다.
2) 질문에 대해 관련 문서를 검색한다.
3) 검색 결과를 바탕으로 LLM이 답변을 생성한다.
4) 답변에 출처를 포함하여 반환한다.

### 작업 정의

Python 비동기 프로그래밍 관련 문서 3개를 사용하여:
- 문서 임베딩 및 저장
- 질문에 대한 검색
- 출처가 포함된 답변 생성

### 필요 라이브러리

- `langchain-openai`: OpenAI 임베딩 및 LLM
- `chromadb`: 벡터 데이터베이스
- `langchain-community`: 문서 로더
- `python-dotenv`: 환경 변수 관리

### 실습 코드

- `practice/chapter8/code/8-6-rag-basic.py`: RAG 시스템 구현
- `practice/chapter8/code/requirements.txt`: 의존성 목록
- `practice/chapter8/data/input/docs/`: 입력 문서

### 실습 산출물(필수)

- `practice/chapter8/data/output/ch08_qa_result.json`: Q&A 결과
- `practice/chapter8/data/output/ch08_index_stats.json`: 인덱스 통계
- `practice/chapter8/data/output/ch08_answer.txt`: 생성된 답변

## 8.7 프로덕션 체크리스트

### 집필 방향

프로덕션 환경에서 RAG 시스템을 운영할 때 고려해야 할 사항을 체크리스트로 정리한다.

### 강화 항목(필수)

- 캐싱 전략 (임베딩 캐싱, 결과 캐싱)
- 비용 최적화 (임베딩 모델 선택, 호출 최소화)
- 평가 지표 (검색 정확도, 답변 품질)
- 모니터링과 로깅

### 표(권장)

**표 8.2** RAG 프로덕션 체크리스트

| 영역 | 체크 항목 |
|-----|---------|
| 검색 | 적절한 Top-K 설정, 필터링 조건 |
| 비용 | 임베딩 캐싱, 배치 처리 |
| 품질 | 정기적 평가, A/B 테스트 |
| 운영 | 로깅, 오류 모니터링 |
