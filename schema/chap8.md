# 제8장: RAG의 기본과 에이전트 메모리 아키텍처 — 집필 계획서

## 장 개요

이 장은 검색 증강 생성(RAG)의 기본 개념과 에이전트 메모리 아키텍처를 다룬다. v2.0에서 에이전트 메모리 분류 체계, 메모리 구현 전략, RAG-메모리 상호보완 관계 섹션을 추가하여, RAG를 넘어선 에이전트 지식 관리의 전체 그림을 조망한다.

### 학습 목표

1) RAG가 필요한 상황과 아키텍처를 이해한다.
2) 에이전트 메모리를 4가지 유형(Working, Episodic, Semantic, Procedural)으로 분류하고 각각의 구현 전략을 설명한다.
3) RAG와 에이전트 메모리의 차이와 상호보완 관계를 판단한다.
4) 답변에 인용/출처를 포함하는 방법을 구현한다.

## 8.7 에이전트 메모리 분류 체계 [신규]

### 집필 방향

인지과학 기반 4가지 메모리 분류(Working, Episodic, Semantic, Procedural)를 에이전트 관점에서 서술한다.

### 강화 항목(반드시 포함)

- Working Memory: 컨텍스트 윈도우, LangGraph State
- Episodic Memory: 과거 이벤트·결과 저장
- Semantic Memory: 일반화된 지식
- Procedural Memory: 학습된 도구 사용 패턴
- 표 8.4: 메모리 4가지 유형 비교

## 8.8 메모리 구현 전략 [신규]

### 집필 방향
