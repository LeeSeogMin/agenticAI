# 제6장: LangGraph 1.0으로 상태·분기·반복을 제어한다 — 집필 계획서

## 장 개요

이 장은 5장에서 구현한 LangChain 에이전트를 확장하여, LangGraph를 사용해 복잡한 워크플로우를 구현한다. v2.0에서 LangGraph 1.0 안정 릴리스, LangSmith 통합, 장기 메모리 섹션을 추가하여 프로덕션 수준의 에이전트 운영을 조망한다.

### 학습 목표

1) LangGraph의 상태(State) 모델을 정의하고 노드 간 데이터를 전달한다.
2) LangGraph 1.0의 안정성 보장과 핵심 변경 사항을 설명한다.
3) 조건부 엣지(conditional edge)를 사용하여 워크플로우 분기를 구현한다.
4) LangSmith를 활용하여 LangGraph 워크플로우를 추적하고 디버깅한다.
5) 체크포인터와 Store API를 구분하여 단기·장기 메모리 전략을 설계한다.

### 장 구성

- 6.1 LangGraph 소개: 그래프 기반 워크플로우의 필요성
- 6.2 상태(State) 모델링
- 6.3 조건 분기: 검증 결과에 따른 경로 선택
- 6.4 반복 루프: 재시도와 검증 루프
- 6.5 실패 복구: 에러 핸들링과 폴백
- 6.6 LangGraph 1.0: 첫 안정 릴리스 [신규]
- 6.7 LangSmith 통합: 워크플로우 추적과 평가 [신규]
- 6.8 LangGraph + 장기 메모리 [신규]
- 6.9 실습: 초안 생성 → 검증 → 수정 워크플로우
- 6.10 실패 사례와 디버깅

## 6.6 LangGraph 1.0: 첫 안정 릴리스 [신규]

### 집필 방향

LangGraph 1.0(2025.10) 릴리스의 의의, API 안정성 보장(v2.0까지), Functional API(@task, @entrypoint) 추가를 서술한다.

### 강화 항목(반드시 포함)

- v2.0까지 브레이킹 체인지 없음 공식 보장
- pre-1.0 vs 1.0 비교 표
- Functional API: @task, @entrypoint 데코레이터

## 6.7 LangSmith 통합 [신규]

### 집필 방향

LangSmith의 중첩 스팬 트레이싱 모델과 평가 워크플로우를 서술한다.

### 강화 항목(반드시 포함)

- 환경 변수 설정만으로 트레이싱 활성화
- @traceable 데코레이터
- 오프라인 평가: 데이터셋 + LLM-as-judge
