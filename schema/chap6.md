# 제6장: LangGraph로 상태·분기·반복을 제어한다 — 집필 계획서

## 장 개요

이 장은 5장에서 구현한 LangChain 에이전트를 확장하여, LangGraph를 사용해 복잡한 워크플로우를 구현한다. 학습자는 (1) 상태(State) 모델링, (2) 조건 분기, (3) 반복(재시도/검증 루프), (4) 실패 복구 패턴을 익힌다. 실습에서는 "초안 생성 → 검증 → 수정" 순환 워크플로우를 구현하고 산출물을 저장한다.

### 학습 목표

1) LangGraph의 상태(State) 모델을 정의하고 노드 간 데이터를 전달한다.
2) 조건부 엣지(conditional edge)를 사용하여 워크플로우 분기를 구현한다.
3) 반복 루프를 구성하여 검증-재시도 패턴을 적용한다.
4) 실패 복구 전략을 설계하고 에러 핸들링을 구현한다.
5) 실습 산출물(워크플로우 실행 로그, 생성된 문서)을 저장하고 해석한다.

---

## 6.1 LangGraph 소개: 그래프 기반 워크플로우의 필요성

### 집필 방향

5장의 ReAct 에이전트는 단순한 도구 호출에 적합하지만, 복잡한 비즈니스 로직(조건 분기, 반복, 병렬 실행)에는 한계가 있다. LangGraph는 노드(node)와 엣지(edge)로 워크플로우를 명시적으로 정의하여 이러한 복잡성을 관리한다.

### 강화 항목(필수)

- ReAct 에이전트 vs 그래프 기반 워크플로우 비교
- LangGraph의 핵심 개념: StateGraph, Node, Edge
- 언제 LangGraph를 사용해야 하는가

### 표(권장)

**표 6.1** ReAct 에이전트와 LangGraph 비교

| 특성 | ReAct 에이전트 | LangGraph |
|-----|--------------|-----------|
| 제어 흐름 | LLM이 결정 | 개발자가 명시적 정의 |
| 조건 분기 | 프롬프트에 의존 | 조건부 엣지로 정확히 제어 |
| 상태 관리 | 메시지 히스토리 | 타입 안전한 State 객체 |
| 디버깅 | 로그 분석 필요 | 그래프 시각화 가능 |

## 6.2 상태(State) 모델링

### 집필 방향

LangGraph의 핵심은 State 객체다. TypedDict를 사용하여 워크플로우 전체에서 공유되는 상태를 정의하고, 각 노드가 상태를 읽고 업데이트하는 방식을 설명한다.

### 강화 항목(필수)

- TypedDict로 State 정의
- Annotated와 operator를 사용한 상태 누적
- 상태 초기화와 최종 상태 추출

### 핵심 코드(3-5줄)

```python
from typing import TypedDict, Annotated
from operator import add

class WorkflowState(TypedDict):
    draft: str
    feedback: Annotated[list[str], add]
    revision_count: int
```

## 6.3 조건 분기: 검증 결과에 따른 경로 선택

### 집필 방향

검증 결과(통과/실패)에 따라 다른 경로로 분기하는 조건부 엣지를 구현한다. 검증 함수가 반환하는 값에 따라 "종료" 또는 "수정" 노드로 이동하는 패턴을 다룬다.

### 강화 항목(필수)

- add_conditional_edges 사용법
- 분기 조건 함수 작성
- 복수 분기 경로 처리

### 핵심 코드(3-5줄)

```python
def should_continue(state: WorkflowState) -> str:
    if state["revision_count"] >= 3:
        return "end"
    return "revise" if state["feedback"] else "end"
```

## 6.4 반복 루프: 재시도와 검증 루프

### 집필 방향

"생성 → 검증 → 실패 시 수정" 사이클을 반복하는 루프를 구현한다. 무한 루프를 방지하기 위한 최대 반복 횟수 설정과 조기 종료 조건을 다룬다.

### 강화 항목(필수)

- 순환 그래프 구성
- 최대 반복 횟수 제한
- 검증 통과 시 조기 종료

## 6.5 실패 복구: 에러 핸들링과 폴백

### 집필 방향

노드 실행 중 에러가 발생했을 때의 처리 전략을 다룬다. 에러를 상태에 기록하고, 폴백 노드로 이동하거나, 사람의 개입을 요청하는 패턴을 설명한다.

### 강화 항목(필수)

- try-except로 노드 내부 에러 처리
- 에러 상태 기록과 복구 경로
- Human-in-the-loop 연결 지점

## 6.6 실습: 초안 생성 → 검증 → 수정 워크플로우

### 실습 목표

1) LangGraph로 순환 워크플로우를 구현한다.
2) OpenAI API를 사용하여 초안을 생성하고, 검증하고, 수정한다.
3) 최대 3회 수정 후 최종 결과를 파일로 저장한다.

### 워크플로우 구성

```
[START] → [generate_draft] → [validate] → {조건 분기}
                                              ↓ (통과)
                                           [END]
                                              ↓ (실패 & 수정 횟수 < 3)
                                           [revise] → [validate] → ...
                                              ↓ (수정 횟수 >= 3)
                                           [END]
```

### 노드 구성

1. **generate_draft**: 주제를 받아 초안 생성
2. **validate**: 초안의 품질을 검증하고 피드백 생성
3. **revise**: 피드백을 반영하여 초안 수정

### 필요 라이브러리

- `langgraph`: 그래프 기반 워크플로우
- `langchain-openai`: OpenAI 통합
- `python-dotenv`: 환경 변수 관리

### 실습 코드

- `practice/chapter6/code/6-6-langgraph-workflow.py`: 워크플로우 구현
- `practice/chapter6/code/requirements.txt`: 의존성 목록

### 실습 산출물(필수)

- `practice/chapter6/data/output/ch06_workflow_log.txt`: 워크플로우 실행 로그

- `practice/chapter6/data/output/ch06_draft_final.txt`: 최종 수정본
