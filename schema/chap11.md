# 제11장: Human-in-the-loop 워크플로우와 점진적 배포 — 집필 계획서

## 장 개요

이 장은 AI 에이전트 시스템에서 사람의 개입이 필요한 지점을 설계하고 구현하는 방법을 다룬다. v2.0에서 점진적 배포 전략(Shadow Mode)과 암호화 서명 기반 감사 로그 설계 섹션을 추가하여, 안전한 에이전트 운영의 전체 그림을 조망한다.

### 학습 목표

1) Human-in-the-loop(HITL)이 필요한 상황을 식별하고, LangGraph의 interrupt와 Command로 승인 워크플로우를 구현한다.
2) Shadow Mode에서 전체 배포까지 5단계 점진적 배포 전략을 설계한다.
3) 암호화 서명과 변조 방지를 포함한 감사 로그 시스템을 설계한다.
4) 고위험 작업을 분리하고 적절한 승인 레벨을 적용한다.

### 장 구성

- 11.1 Human-in-the-loop의 필요성
- 11.2 HITL 패턴 분류
- 11.3 LangGraph의 interrupt와 Command
- 11.4 승인 요청 UX 설계
- 11.5 감사 로그와 승인 이력
- 11.6 고위험 작업 분리
- 11.7 점진적 배포 전략 [신규]
- 11.8 감사 로그 설계: 무결성과 추적 가능성 [신규]
- 11.9 실습: 승인 워크플로우 구현
- 11.10 실패 사례와 교훈

## 11.7 점진적 배포 전략 [신규]

### 집필 방향

에이전트 시스템의 점진적 배포 전략을 5단계(Shadow Mode → 샌드박스 → 스테이징 → 소규모 프로덕션 → 전체 배포)로 서술한다.

### 강화 항목(반드시 포함)

- 표 11.3: 점진적 배포 5단계 비교
- Shadow Mode: 분석만 수행, 실행 안 함, 기존 시스템과 결과 비교
- 카나리 배포와 자동 롤백 메커니즘
- 에이전트 비결정적 특성과 배포 리스크

---

## 11.8 감사 로그 설계: 무결성과 추적 가능성 [신규]

### 집필 방향

프로덕션 환경에서 요구되는 암호화 서명, 해시 체인 기반 변조 방지, 에이전트 행동의 완전한 추적 가능성을 서술한다.

### 강화 항목(반드시 포함)

- HMAC 기반 암호화 서명으로 로그 무결성 보장
- 해시 체인(블록체인 원리): 삭제/변경 즉시 탐지
- 표 11.4: 에이전트 감사 로그 확장 스키마 (요청·추론·도구·승인·결과 5계층)
- 완전한 추적 가능성: 규제 대응, 사고 조사, 시스템 개선
