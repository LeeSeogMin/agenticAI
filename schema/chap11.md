# 제11장: Human-in-the-loop 워크플로우 설계 — 집필 계획서

## 장 개요

이 장은 AI 에이전트 시스템에서 사람의 개입이 필요한 지점을 설계하고 구현하는 방법을 다룬다. 학습자는 (1) HITL이 필요한 상황 식별, (2) 승인 요청 UX 설계, (3) 로그/감사 추적, (4) 고위험 작업 분리를 익힌다. 실습에서는 승인 없이는 실행되지 않는 단계를 포함한 워크플로우를 구현한다.

### 학습 목표

1) Human-in-the-loop(HITL)이 필요한 상황을 식별한다.
2) LangGraph의 interrupt와 Command를 활용한 승인 워크플로우를 구현한다.
3) 감사 로그와 승인 이력을 추적하는 시스템을 설계한다.
4) 고위험 작업을 분리하고 적절한 승인 레벨을 적용한다.
5) 승인 없이는 실행되지 않는 워크플로우를 구축한다.

---

## 11.1 Human-in-the-loop의 필요성

### 집필 방향

AI 시스템에서 사람의 개입이 필요한 이유와 상황을 설명한다. 10장의 검증 파이프라인에서 에스컬레이션된 경우를 자연스럽게 연결한다.

### 강화 항목(필수)

- HITL의 정의와 목적
- AI 자율성과 사람 통제의 균형
- HITL이 필요한 상황 (고위험, 불확실성, 윤리적 판단)
- 10장 검증 파이프라인과의 연결

---

## 11.2 HITL 패턴 분류

### 집필 방향

다양한 HITL 패턴을 분류하고 각각의 적용 시나리오를 설명한다.

### 강화 항목(필수)

- Approval (승인): 실행 전 명시적 허가
- Review (검토): 결과 확인 후 진행
- Edit (수정): 상태나 결과를 직접 수정
- Escalation (에스컬레이션): 상위 권한자에게 위임
- Override (오버라이드): AI 결정을 사람이 덮어씀

### 표(권장)

**표 11.1** HITL 패턴별 적용 시나리오

| 패턴 | 설명 | 적용 예시 |
|-----|------|---------|
| Approval | 실행 전 승인 | 결제, 이메일 발송 |
| Review | 결과 확인 | 문서 요약, 보고서 생성 |
| Edit | 상태 수정 | 프롬프트 수정, 데이터 보정 |
| Escalation | 권한 위임 | 고액 결제, 민감 정보 접근 |
| Override | 결정 덮어씀 | 자동 분류 결과 수정 |

## 11.6 고위험 작업 분리

### 집필 방향

작업을 위험도에 따라 분류하고 적절한 승인 레벨을 적용하는 방법을 다룬다.

### 강화 항목(필수)

- 위험도 분류 기준 (되돌릴 수 없음, 비용, 외부 영향)
- 승인 레벨 정의 (자동, 단일 승인, 다중 승인)
- 권한 기반 접근 제어
- 긴급 우회 절차

### 표(권장)

**표 11.2** 위험도별 승인 레벨

| 위험도 | 특성 | 승인 레벨 | 예시 |
|:---:|-----|---------|-----|
| 낮음 | 되돌릴 수 있음, 내부 영향 | 자동 승인 | 로그 조회, 검색 |
| 중간 | 일부 되돌릴 수 있음 | 단일 승인 | 파일 수정, API 호출 |
| 높음 | 되돌릴 수 없음, 외부 영향 | 다중 승인 | 결제, 데이터 삭제 |

## 11.7 실습: 승인 워크플로우 구현

### 실습 목표

1) LangGraph의 interrupt를 사용한 승인 워크플로우를 구현한다.
2) 위험도에 따른 차등 승인 로직을 적용한다.
3) 감사 로그를 기록하고 저장한다.
4) 승인 없이는 고위험 작업이 실행되지 않음을 검증한다.

### 작업 정의

문서 처리 시스템을 구현하여:
- 저위험 작업 (조회): 자동 진행
- 중위험 작업 (수정): 단일 승인 필요
- 고위험 작업 (삭제/외부 전송): 반드시 승인 후 실행

### 필요 라이브러리

- `langgraph`: 워크플로우 구성
- `langchain-openai`: OpenAI LLM
- `python-dotenv`: 환경 변수

### 실습 코드

- `practice/chapter11/code/11-7-hitl-workflow.py`: HITL 워크플로우 구현
- `practice/chapter11/code/requirements.txt`: 의존성 목록

### 실습 산출물(필수)

- `practice/chapter11/data/output/ch11_hitl_result.json`: 워크플로우 실행 결과
- `practice/chapter11/data/output/ch11_audit_log.json`: 감사 로그
