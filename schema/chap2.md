## 2.1 왜 재현 가능한 환경이 중요한가

### 집필 방향
AI 협업 개발에서 재현 가능한 환경의 중요성을 실제 실패 사례와 함께 설명한다. "내 컴퓨터에서는 되는데..."라는 흔한 문제의 원인과 해결 방향을 제시한다.

### 내용 구성

**2.1.1 AI 코드 생성의 환경 의존성 문제**
- AI가 생성한 코드가 환경에 따라 다르게 동작하는 사례
- 라이브러리 버전, 운영체제, 파일 경로 등 숨겨진 의존성
- 재현 불가능한 코드의 비용: 디버깅 시간, 협업 어려움

**2.1.2 재현 가능한 실행의 핵심 요소**
- 격리된 실행 환경 (가상환경)
- 고정된 의존성 (requirements.txt, poetry.lock 등)
- 일관된 경로 처리 (상대 경로, pathlib)
- 버전 관리와 환경 설정 분리

## 2.2 Python 가상환경 완전 정복

### 집필 방향
venv, conda 등 주요 가상환경 도구의 특징을 비교하고, 실무에서 가장 범용적인 venv 사용법을 상세히 다룬다.

### 내용 구성

**2.2.1 가상환경이란 무엇인가**
- 시스템 Python과 프로젝트 Python의 분리
- 의존성 충돌 방지의 원리
- 프로젝트별 격리의 장점

**2.2.2 venv로 가상환경 만들기**
- macOS/Linux: `python3 -m venv venv && source venv/bin/activate`
- Windows: `python -m venv venv && venv\Scripts\activate`
- 가상환경 활성화 확인: `which python`, `where python`

**2.2.3 conda vs venv: 언제 무엇을 선택할까**
- venv: 순수 Python 프로젝트, 가볍고 빠름, 표준 라이브러리 포함
- conda: 데이터 과학, 비Python 의존성(CUDA 등), 환경 복제 용이
- 선택 기준표 제공

## 2.3 의존성 관리와 버전 고정

### 집필 방향
requirements.txt를 중심으로 의존성 관리 방법을 설명하고, 버전 고정의 중요성을 실제 사례로 보여준다.

### 내용 구성

**2.3.1 requirements.txt 작성법**
- `pip freeze > requirements.txt`의 문제점 (불필요한 패키지 포함)
- 수동 관리 vs 자동 생성
- 버전 지정 방식: `==`, `>=`, `~=`의 차이

**2.3.2 의존성 버전 고정 전략**
- 개발 시: 유연한 버전 (`>=1.0`)
- 배포 시: 엄격한 버전 (`==1.0.0`)
- lock 파일의 역할 (pip-tools, poetry)

**2.3.3 의존성 충돌 해결**
- 충돌 발생 시 진단 방법
- pip의 의존성 해결 알고리즘
- 실제 충돌 해결 사례

## 2.4 크로스 플랫폼 경로 처리

### 집필 방향
Windows와 macOS/Linux에서 동일하게 동작하는 코드를 작성하는 방법을 다룬다. pathlib 사용을 표준으로 권장한다.

### 내용 구성

**2.4.1 플랫폼별 경로 차이**
- Windows: 백슬래시 `\`, 드라이브 문자 `C:`
- macOS/Linux: 슬래시 `/`, 루트 `/`
- 하드코딩의 위험성

**2.4.2 pathlib으로 경로 다루기**
- `Path(__file__).parent`로 스크립트 기준 상대 경로
- `/` 연산자로 경로 조합: `base_path / "data" / "input"`
- 존재 확인, 생성, 읽기/쓰기

**2.4.3 os.path vs pathlib 비교**
- os.path: 문자열 기반, 레거시 코드 호환
- pathlib: 객체 기반, 가독성 좋음, Python 3.4+ 표준
- 마이그레이션 가이드

### 코드 예제

## 2.5 실습: "실행-검증-산출물 저장" 템플릿 만들기

### 실습 목표
- 재현 가능한 실행 환경에서 동작하는 표준 스크립트 템플릿 작성
- 실행 결과를 검증하고 산출물을 구조화된 경로에 저장

### 실습 구성

**Step 1: 프로젝트 구조 생성**
```
practice/chapter2/
├── code/
│   ├── 2-5-template.py      # 표준 템플릿
│   ├── requirements.txt     # 의존성 목록
│   └── README.md            # 실행 가이드
└── data/
    ├── input/               # 입력 데이터
    └── output/              # 산출물 저장
```

**Step 2: 표준 템플릿 구현**
- 환경 검증 함수: Python 버전, 의존성 확인
- 경로 설정 함수: 입력/출력 경로 자동 구성
- 실행 및 검증 함수: 결과값 검증 로직
- 산출물 저장 함수: 타임스탬프 포함 파일명

**Step 3: 실제 실행 및 검증**
- 가상환경에서 실행
- 실행 결과 확인
- 산출물 파일 생성 확인

### 필요 라이브러리
- pathlib (표준)
- datetime (표준)
- json (표준)
- sys (표준)

### 예상 결과물
- `practice/chapter2/code/2-5-template.py` - 표준 템플릿 스크립트
- `practice/chapter2/data/output/result_YYYYMMDD_HHMMSS.json` - 실행 결과
