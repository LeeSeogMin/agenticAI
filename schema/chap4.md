# 제4장: 첫 MCP 서버 만들기: 외부 API 래핑과 A2A 개요 — 집필 계획서

## 장 개요

이 장은 3장에서 설계한 도구 명세를 바탕으로, 실제 외부 API를 MCP 서버로 래핑하는 과정을 다룬다. v2.0에서 OAuth 2.1 인증, A2A 프로토콜 개요, 프로토콜 선택 기준 섹션을 추가하여, MCP 서버의 인증 체계와 에이전트 간 통신 프로토콜 전경을 함께 조망한다.

### 학습 목표

1) 외부 API를 MCP 서버로 래핑할 때 고려할 사항(인증/실패/로깅/테스트)을 설명한다.
2) OAuth 2.1 기반 원격 MCP 서버 인증의 흐름과 핵심 개념을 이해한다.
3) A2A(Agent-to-Agent) 프로토콜의 개념과 MCP와의 관계를 설명한다.
4) MCP, A2A, 직접 API 호출 중 상황에 맞는 프로토콜을 선택할 수 있다.

### 장 구성

- 4.1 외부 API 래핑의 목표와 고려사항
- 4.2 인증키 관리: .env와 환경 변수
- 4.3 실패 처리: 타임아웃, 재시도, 에러 응답
- 4.4 로깅: 요청과 응답을 추적 가능하게
- 4.5 테스트 가능한 구조: 의존성 분리
- 4.6 OAuth 2.1 인증: 원격 MCP 서버의 인증 프레임워크 [신규]
- 4.7 A2A(Agent-to-Agent) 프로토콜 개요 [신규]
- 4.8 프로토콜 선택 기준: MCP vs A2A vs 직접 API 호출 [신규]
- 4.9 실습: OpenWeatherMap API를 MCP 서버로 래핑

## 4.6 OAuth 2.1 인증 [신규]

### 집필 방향

OAuth 2.1 도입 배경(March 2025), 리소스/인가 서버 분리(June 2025), PKCE 필수화(November 2025)의 진화를 서술한다.

### 강화 항목(반드시 포함)

- RFC 9728 (Protected Resource Metadata)
- RFC 8707 (Resource Indicators)
- PKCE 필수, CIMD (Client ID Metadata Document)

### 집필 방향
