# 제4장: 첫 MCP 서버 만들기: 외부 API 래핑 — 집필 계획서

## 장 개요

이 장은 3장에서 설계한 도구 명세를 바탕으로, 실제 외부 API를 MCP 서버로 래핑하는 과정을 다룬다. 학습자는 (1) 인증키 처리(.env), (2) 실패 처리(타임아웃/재시도), (3) 로깅, (4) 테스트 가능한 구조를 직접 구현하며, "운영 가능한 MCP 서버"의 최소 요건을 이해한다. 실습에서는 OpenWeatherMap API를 MCP 서버로 래핑하고, 실제 호출 결과를 파일로 저장한다.

### 학습 목표

1) 외부 API를 MCP 서버로 래핑할 때 고려할 사항(인증/실패/로깅/테스트)을 설명한다.
2) `.env` 파일로 인증키를 분리하고, 코드에 하드코딩하지 않는 패턴을 적용한다.
3) 타임아웃, 재시도, 에러 응답 형식을 포함한 실패 처리 전략을 구현한다.
4) 로깅을 통해 요청/응답을 추적 가능하게 만들고, 테스트 가능한 구조로 설계한다.
5) 실습 산출물(호출 결과 JSON, 로그 파일)을 저장하고 해석한다.

---

## 4.1 외부 API 래핑의 목표와 고려사항

### 집필 방향

MCP 서버로 외부 API를 래핑하는 것은 단순히 "호출을 감싸는 것"이 아니라, 에이전트가 안정적으로 사용할 수 있는 "계약(contract)"을 만드는 것임을 강조한다. 3장의 도구 명세가 "무엇을 할 것인가"였다면, 4장은 "어떻게 안전하게 할 것인가"를 다룬다.

### 강화 항목(필수)

- 외부 API 호출 시 발생할 수 있는 문제(네트워크 오류, 인증 실패, 레이트 리밋, 응답 지연)를 유형별로 정리
- 각 문제에 대한 대응 전략(재시도, 폴백, 에러 응답)을 표로 제시

### 표(권장)

**표 4.1** 외부 API 호출 시 문제 유형과 대응 전략

| 문제 유형 | 원인 | 대응 전략 |
|----------|------|----------|
| 네트워크 오류 | 연결 실패, DNS 오류 | 재시도(지수 백오프) |
| 인증 실패 | 잘못된 키, 만료된 키 | 명확한 에러 메시지, 키 검증 |
| 레이트 리밋 | 요청 과다 | 대기 후 재시도, 캐싱 |
| 응답 지연 | 서버 과부하 | 타임아웃 설정, 취소 |

## 4.2 인증키 관리: .env와 환경 변수

### 집필 방향

API 키를 코드에 하드코딩하면 발생하는 보안 문제(저장소 노출, 로그 노출)를 설명하고, `.env` 파일과 `python-dotenv`를 활용한 분리 패턴을 소개한다. 환경 변수 우선순위(시스템 → .env → 기본값)도 다룬다.

### 핵심 코드(3-5줄)

```python
from dotenv import load_dotenv
import os

load_dotenv()
api_key = os.getenv("OPENWEATHERMAP_API_KEY")
```

### 강화 항목(필수)

- `.env` 파일은 `.gitignore`에 반드시 추가
- `.env.example` 파일로 필요한 변수 목록을 공유하는 패턴

## 4.3 실패 처리: 타임아웃, 재시도, 에러 응답

### 집필 방향

외부 API는 언제든 실패할 수 있으므로, "성공 경로"뿐 아니라 "실패 경로"도 설계해야 함을 강조한다. 타임아웃 설정, 재시도 전략(최대 횟수, 지수 백오프), 에러 응답 형식(MCP 표준)을 다룬다.

### 강화 항목(필수)

- 재시도 가능한 오류와 불가능한 오류의 구분
- 지수 백오프(exponential backoff) 개념과 구현
- MCP 에러 응답 형식 준수

### 핵심 코드(3-5줄)

```python
import time

for attempt in range(max_retries):
    try:
        response = requests.get(url, timeout=10)
        response.raise_for_status()
        break
    except requests.RequestException:
        time.sleep(2 ** attempt)
```

## 4.4 로깅: 요청과 응답을 추적 가능하게

### 집필 방향

운영 환경에서 문제 발생 시 원인을 파악하려면 로깅이 필수임을 설명한다. Python `logging` 모듈을 활용해 요청/응답/에러를 파일로 기록하는 패턴을 소개한다. 민감 정보(API 키, 사용자 데이터)는 로그에서 마스킹해야 함을 강조한다.

### 강화 항목(필수)

- 로그 레벨(DEBUG, INFO, WARNING, ERROR) 활용
- 로그 파일 경로는 상대 경로로 설정
- 민감 정보 마스킹 패턴

### 핵심 코드(3-5줄)

```python
import logging

logging.basicConfig(
    filename="logs/mcp_server.log",
    level=logging.INFO,
    format="%(asctime)s - %(levelname)s - %(message)s"
)
```

## 4.5 테스트 가능한 구조: 의존성 분리

### 집필 방향

외부 API에 의존하는 코드는 테스트하기 어렵다. API 호출 로직을 분리하여 모킹(mocking) 가능한 구조로 만드는 패턴을 소개한다. 실제 API 호출 없이도 로직을 검증할 수 있음을 보여준다.

### 강화 항목(필수)

- 함수/클래스 수준에서 API 클라이언트를 주입받는 패턴
- `unittest.mock`을 활용한 간단한 테스트 예시

## 4.6 실습: OpenWeatherMap API를 MCP 서버로 래핑

### 실습 목표

1) OpenWeatherMap API를 호출하는 MCP 서버를 구현한다.
2) `.env`로 API 키를 관리하고, 타임아웃/재시도/로깅을 포함한다.
3) 실제 호출 결과를 JSON 파일로 저장한다.

### API 선택 이유

OpenWeatherMap은 무료 티어를 제공하고, API 키 발급이 간단하며, 요청/응답 구조가 명확하다. 날씨 정보는 실시간 데이터이므로 "실제 실행 결과"를 명확히 확인할 수 있다.

### 실습 구성

1. **환경 설정**: `.env` 파일에 `OPENWEATHERMAP_API_KEY` 설정
2. **MCP 서버 구현**: 도구 정의, 요청 처리, 응답 반환
3. **실패 처리**: 타임아웃(10초), 재시도(최대 3회), 에러 응답
4. **로깅**: 요청/응답을 `logs/` 디렉토리에 기록
5. **테스트**: 간단한 유닛 테스트로 로직 검증
6. **결과 저장**: 호출 결과를 `data/output/`에 JSON으로 저장

### 필요 라이브러리

- `mcp`: MCP Python SDK
- `python-dotenv`: 환경 변수 관리
- `httpx` 또는 `requests`: HTTP 클라이언트
- `logging`: 표준 라이브러리

### 실습 코드

- `practice/chapter4/code/4-6-weather-mcp-server.py`: MCP 서버 구현
- `practice/chapter4/code/4-6-test-server.py`: 테스트 클라이언트
- `practice/chapter4/code/requirements.txt`: 의존성 목록

### 실습 산출물(필수)

- `practice/chapter4/data/output/ch04_weather_response.json`: 실제 API 호출 결과
- `practice/chapter4/data/output/ch04_server_log.txt`: 서버 로그
- `practice/chapter4/.env.example`: 환경 변수 템플릿
