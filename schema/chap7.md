# 제7장: 멀티에이전트 시스템: 프레임워크 비교와 A2A 실전 — 집필 계획서

## 장 개요

이 장은 멀티에이전트 시스템의 개념과 필요성을 소개하고, 다양한 프레임워크의 특징과 선택 기준을 다룬다. v2.0에서 프레임워크 비교를 5종으로 확장하고, A2A 실전 연동, 로우코드/노코드 대안 섹션을 추가하여 실무 선택지를 넓힌다.

### 학습 목표

1) 멀티에이전트 시스템이 필요한 상황을 식별한다.
2) 주요 프레임워크(LangGraph, OpenAI Agents SDK, Google ADK, CrewAI, AutoGen)의 설계 철학과 차이를 비교한다.
3) A2A 프로토콜을 활용하여 이종 프레임워크 에이전트를 연동하는 방법을 설명한다.
4) 로우코드/노코드 도구와 코드 기반 프레임워크의 선택 기준을 판단한다.

## 7.3 프레임워크 비교 [신규/강화]

### 집필 방향

기존 3종(CrewAI, AutoGen, LangGraph)에 OpenAI Agents SDK, Google ADK를 추가하여 5종 비교. AutoGen v0.4/AG2 포크 상황도 반영.

### 강화 항목(반드시 포함)

- Google ADK: SequentialAgent, ParallelAgent, LoopAgent, sub_agents
- OpenAI Agents SDK: 핸드오프 메커니즘 (transfer_to)
- AutoGen v0.4: 비동기 이벤트 기반 액터 모델, Magentic-One, AG2 포크
- 표 7.2: 5종 프레임워크 비교 표

---

## 7.5 A2A 실전 [신규]

### 집필 방향

A2A 프로토콜의 실전 적용: 에이전트 카드 작성, 태스크 위임 흐름, 이종 프레임워크 연동을 서술한다.

### 강화 항목(반드시 포함)

- 에이전트 카드 JSON 구조와 필수 필드
- SendMessage → Task 상태 추적 흐름
- MCP(도구) + A2A(에이전트 간) 상호보완

---

## 7.6 로우코드/노코드 대안 [신규]

### 집필 방향

n8n, Dify 등 로우코드 도구를 소개하고, 코드 기반 프레임워크와의 선택 기준을 제시한다.

### 강화 항목(반드시 포함)

- n8n (17만+ stars): 범용 자동화, MCP 지원
- Dify (12만+ stars): AI 네이티브 앱 플랫폼
- 표 7.3: 코드 기반 vs 로우코드/노코드
