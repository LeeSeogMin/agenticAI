# 제7장: 멀티에이전트 개요와 선택 전략 — 집필 계획서

## 장 개요

이 장은 멀티에이전트 시스템의 개념과 필요성을 소개하고, 다양한 프레임워크(CrewAI, AutoGen, LangGraph)의 특징과 선택 기준을 다룬다. 학습자는 (1) 단일 에이전트와 멀티에이전트의 차이, (2) 멀티에이전트 아키텍처 패턴, (3) Human-in-the-loop 설계, (4) 프레임워크 선택 기준을 익힌다. 실습에서는 동일 과제를 단일 에이전트와 멀티에이전트로 각각 구현하여 차이를 비교한다.

### 학습 목표

1) 멀티에이전트 시스템이 필요한 상황을 식별한다.
2) 주요 멀티에이전트 아키텍처 패턴(계층형, 협력형, 경쟁형)을 이해한다.
3) CrewAI, AutoGen, LangGraph의 특징과 차이점을 비교한다.
4) Human-in-the-loop 설계 원칙을 적용한다.
5) 동일 과제를 단일/멀티에이전트로 구현하여 트레이드오프를 분석한다.

---

## 7.1 멀티에이전트 시스템이란?

### 집필 방향

단일 에이전트의 한계(컨텍스트 제한, 전문성 부족, 복잡성 증가)를 설명하고, 멀티에이전트 시스템이 이를 해결하는 방식을 소개한다. 각 에이전트가 전문 역할을 맡아 협력하는 개념을 설명한다.

### 강화 항목(필수)

- 단일 에이전트 vs 멀티에이전트 비교
- 멀티에이전트가 효과적인 시나리오
- 멀티에이전트의 복잡성과 비용 고려

### 표(권장)

**표 7.1** 단일 에이전트 vs 멀티에이전트

| 특성 | 단일 에이전트 | 멀티에이전트 |
|-----|-------------|-------------|
| 복잡성 | 낮음 | 높음 |
| 전문성 | 일반적 | 역할별 전문화 |
| 컨텍스트 관리 | 단순 | 분산/공유 |
| 비용 | 낮음 | 높음 (API 호출 증가) |
| 적합한 작업 | 단순 Q&A, 도구 호출 | 복잡한 워크플로우, 다단계 분석 |

---

## 7.2 멀티에이전트 아키텍처 패턴

### 집필 방향

멀티에이전트 시스템의 주요 아키텍처 패턴을 소개한다. 각 패턴의 특징, 장단점, 적합한 사용 사례를 설명한다.

### 강화 항목(필수)

- 계층형(Hierarchical): 관리자 에이전트가 작업을 할당
- 협력형(Collaborative): 에이전트들이 동등하게 협력
- 경쟁형(Adversarial): 에이전트들이 검증/반박
- 순차형(Sequential): 파이프라인 형태로 처리

---

## 7.3 프레임워크 비교: CrewAI, AutoGen, LangGraph

### 집필 방향

각 프레임워크의 설계 철학, 핵심 기능, 장단점을 비교한다. 프레임워크 선택 시 고려해야 할 기준을 제시한다.

### 강화 항목(필수)

- CrewAI: 역할 기반 에이전트, 간단한 협업
- AutoGen: Microsoft 개발, 대화 기반 멀티에이전트
- LangGraph: 그래프 기반 워크플로우, 세밀한 제어

### 표(권장)

**표 7.2** 멀티에이전트 프레임워크 비교

| 기준 | CrewAI | AutoGen | LangGraph |
|-----|--------|---------|-----------|
| 설계 철학 | 역할 기반 팀 | 대화 기반 협업 | 그래프 워크플로우 |
| 학습 곡선 | 낮음 | 중간 | 높음 |
| 제어 수준 | 추상화 높음 | 중간 | 세밀한 제어 |
| Human-in-loop | 기본 지원 | 강력 지원 | 커스텀 구현 |
| 적합한 사용 | 빠른 프로토타입 | 대화형 협업 | 복잡한 워크플로우 |

## 7.4 Human-in-the-loop 설계

### 집필 방향

사람의 개입이 필요한 지점을 식별하고, 승인/거부 메커니즘을 설계하는 방법을 설명한다. 고위험 작업에서의 Human-in-the-loop 필요성을 강조한다.

### 강화 항목(필수)

- 승인이 필요한 작업 유형 식별
- 승인 요청 UX 설계
- 타임아웃과 폴백 처리

## 7.5 실습: 단일 에이전트 vs 멀티에이전트 비교

### 실습 목표

1) 동일한 작업(기술 문서 작성)을 단일 에이전트로 구현한다.
2) 같은 작업을 멀티에이전트(연구자, 작성자, 검토자)로 구현한다.
3) 두 접근법의 결과물, 실행 시간, API 호출 수를 비교한다.

### 작업 정의

"Python 비동기 프로그래밍"에 대한 기술 문서 작성:
- 개념 설명
- 코드 예시
- 모범 사례

### 에이전트 구성 (멀티에이전트)

1. **연구자(Researcher)**: 주제 조사, 핵심 개념 추출
2. **작성자(Writer)**: 연구 결과를 바탕으로 문서 작성
3. **검토자(Reviewer)**: 문서 품질 검토, 피드백 제공

### 필요 라이브러리

- `langchain-openai`: OpenAI 통합
- `langgraph`: 워크플로우 (멀티에이전트용)
- `python-dotenv`: 환경 변수 관리

### 실습 코드

- `practice/chapter7/code/7-5-single-agent.py`: 단일 에이전트 구현
- `practice/chapter7/code/7-5-multi-agent.py`: 멀티에이전트 구현
- `practice/chapter7/code/requirements.txt`: 의존성 목록

### 실습 산출물(필수)

- `practice/chapter7/data/output/ch07_single_result.json`: 단일 에이전트 결과
- `practice/chapter7/data/output/ch07_multi_result.json`: 멀티에이전트 결과
- `practice/chapter7/data/output/ch07_comparison.json`: 비교 분석
- `practice/chapter7/data/output/ch07_document_single.txt`: 단일 에이전트 생성 문서
- `practice/chapter7/data/output/ch07_document_multi.txt`: 멀티에이전트 생성 문서
