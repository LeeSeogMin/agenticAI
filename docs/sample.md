# 원고 작성 표준 (sample)

이 문서는 `docs/ch{N}.md` 원고를 작성할 때의 **표준 형식**을 정의한다. 본문에 포함되는 수치·표·산출물 목록·실행 결과는 반드시 실제 실행/생성으로 검증된 것만 사용한다.

---

## 1. 문체·서술 규칙

본문은 학술적이되 대중 교재에 맞게 과도한 전문 용어를 피하고, 필요한 경우 첫 등장 시 영문 병기를 포함한다(예: “모델 컨텍스트 프로토콜(Model Context Protocol, MCP)”).

문장은 격식체 평서문(“…이다/…한다”)을 기본으로 하며, 본문에서는 개조식을 피한다. 다만 학습목표, 핵심정리, 표, 코드 블록은 예외로 한다.

---

## 2. 장/절 구조 기본

원고 파일명은 `docs/ch{N}.md`이며, 최소 구성은 다음을 권장한다.

1) 장 도입: 독자가 “왜 필요한가”를 이해하도록 문제 상황과 목표를 설명한다.  
2) 개념: 핵심 개념을 정의하고 오해 포인트를 정리한다.  
3) 절차/방법: 실제 적용 절차를 단계적으로 설명한다.  
4) 실습: `practice/chapter{N}/code/`의 실행 가능한 스크립트로 연결한다.  
5) 정리: 핵심정리(짧은 리스트 가능)와 다음 장 연결을 제공한다.

---

## 3. 코드 제시 원칙 (핵심만 3–5줄)

본문에는 **핵심 코드만 3–5줄** 포함한다. 전체 실행 코드는 항상 `practice/` 아래에 두고, 본문에는 아래 형태로만 참조한다.

_전체 코드는 practice/chapter{N}/code/{파일명}.py 참고_

예시는 다음처럼 “핵심 개념을 대표하는 최소 코드”만 보여준다.

```python
from pathlib import Path

output_dir = Path("data") / "output"
output_dir.mkdir(parents=True, exist_ok=True)
```

---

## 4. 경로·환경·명령어 (크로스 플랫폼)

본문/코드에서 플랫폼 종속 경로(예: `/Users/...`)를 하드코딩하지 않는다. 파이썬 코드는 `pathlib.Path` 또는 `os.path.join()`을 사용하고, 문서의 실행 안내는 프로젝트 루트 기준 상대 경로로 작성한다.

실습 실행 안내의 기본 형태는 다음을 따른다.

```bash
cd practice/chapter{N}
python3 -m venv venv
source venv/bin/activate  # macOS/Linux
# venv\Scripts\activate   # Windows
pip install -r code/requirements.txt
python3 code/{N}-{M}-{주제}.py
```

---

## 5. 표·그림·산출물 표기

표 제목은 표 위에, 그림 제목은 그림 아래에 둔다. 번호는 `{장번호}.{순번}` 체계를 사용한다.

- 표 제목 예: `**표 2.1** 도구 비교 요약`
- 그림 제목 예: `**그림 3.2** 워크플로우 개요`

산출물은 기본적으로 `practice/chapter{N}/data/output/` 아래에 저장하며, 본문에는 **실제 생성된 경로**를 그대로 적는다.

---

## 6. “실제 실행 결과만” 규칙 (필수)

다음은 본문에 절대 쓰지 않는다.

1) 실행하지 않은 결과를 “예시 출력”으로 제시하는 것  
2) 측정하지 않은 성능 수치/비교표/그래프를 임의로 작성하는 것  
3) 존재하지 않는 파일/폴더/산출물을 만든 것처럼 기재하는 것

---

## 7. 참고문헌

허구의 참고문헌은 금지한다. 인용은 검증 가능한 실재 문헌만 사용하며, 형식은 다음을 따른다.

저자명. (연도). 논문제목. *저널명*. URL/DOI
