# 강의계획서

## GitHub Copilot으로 AI 에이전트 만들기

---

## 강의 정보

| 항목 | 내용 |
|------|------|
| **과목명** | GitHub Copilot으로 AI 에이전트 만들기 |
| **유형** | 정규 교과목 |
| **학기** | 2026년 1학기 |
| **주차** | 15주 |
| **주당 시간** | 3시간 (주 2회, 회당 90분) |
| **수업 방식** | 1회차 강의 + 2회차 실습/과제 |
| **대상** | 기본 프로그래밍 경험이 있는 학부생 |
| **선수 조건** | Python 또는 JavaScript 기초 문법, VS Code 사용 경험 |
| **준비물** | 개인 노트북 (Windows/macOS/Linux) |

---

## 강의 목표

본 과목을 이수하면 다음 역량을 갖추게 된다.

1. GitHub Copilot의 에이전트 모드를 활용하여 자연어 명세로 소프트웨어를 생성할 수 있다
2. MCP의 아키텍처를 이해하고 기존 서버를 연결하여 Copilot의 기능을 확장할 수 있다
3. MCP 서버를 직접 설계·구현하고 테스트·배포할 수 있다
4. Skills 파일로 Copilot에 도메인 지식과 절차를 주입할 수 있다
5. MCP + Skills를 조합하여 업무 자동화 에이전트를 설계·구현할 수 있다
6. Coding Agent를 활용하여 이슈→PR→리뷰 팀 워크플로우를 자동화할 수 있다

---

## 수업 운영

매주 2회 수업으로 구성하며, 회차별 역할은 다음과 같다.

| 회차 | 시간 | 내용 | 비고 |
|------|------|------|------|
| **1회차** | 90분 | 이론 강의 + 라이브 시연 | 개념·원리·아키텍처 중심 |
| **2회차** | 90분 | 실습 + 과제 제출 | Copilot 활용, 산출물 제출 |

## 수강 전 준비

수강생은 1주차 시작 전까지 아래 환경을 갖추어야 한다.

| 항목 | 설명 | 비고 |
|------|------|------|
| VS Code | 최신 버전 (1.102+), Insiders 권장 | [다운로드](https://code.visualstudio.com/) |
| GitHub 계정 | Student Developer Pack 인증 → Copilot Pro 무료 | [등록 안내](https://education.github.com/pack) |

| Python 3.10+ | MCP 서버 개발용 | `python3 --version` |
| Node.js 18+ | MCP 서버 실행용 (`npx`) | `node --version` |
| Git | 기본 사용법 숙지 | `git --version` |

> Copilot Pro 등록이 어려운 경우 `깃허브.md` 가이드를 참고한다.

## 강의 흐름

```
Phase 1. 기초 (1~4주)
[1] 환경 구축 ─── AI 코딩 도구 전경을 파악하고 실습 환경을 완성한다
       │
[2] Agent 기초 ── Copilot Agent Mode로 자연어 개발을 체험한다
       │
[3] Agent 심화 ── 한계를 인식하고 효과적인 활용 전략을 세운다
       │
[4] MCP 개념 ─── MCP 아키텍처와 생태계를 이해한다

Phase 2. MCP (5~7주)
[5] MCP 연결 ─── 기존 MCP 서버를 연결하여 Copilot을 확장한다
       │
[6] MCP 구현 ─── Python SDK로 MCP 서버를 직접 만든다
       │
[7] MCP 심화 ─── 에러 처리, 테스트, 배포까지 고려한다

[8] ────────── 중간고사 ──────────

Phase 3. Skills & 통합 (9~11주)
[9]  Skills 기초 ─ SKILL.md 구조를 익히고 기본 스킬을 작성한다
       │
[10] Skills 심화 ─ 설계 패턴을 적용하고 도메인 특화 스킬을 만든다
       │
[11] 통합 프로젝트  MCP + Skills로 자동화 에이전트를 완성한다

Phase 4. 팀 & 프로젝트 (12~15주)
[12] Coding Agent ─ 이슈→PR→리뷰 팀 워크플로우를 자동화한다
       │
[13] 프로젝트 설계 ─ 최종 프로젝트를 체계적으로 설계한다
       │
[14] 프로젝트 개발 ─ 구현을 완성하고 발표를 준비한다
       │
[15] 기말고사 + 발표
```
